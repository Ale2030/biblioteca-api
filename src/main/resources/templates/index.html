Sistema de Biblioteca Digital - API v1.0

Bienvenido a la API del Sistema de Biblioteca Digital. Esta API permite gestionar libros, autores y editoriales con endpoints REST para realizar operaciones CRUD (Crear, Leer, Actualizar y Eliminar).

Entidades Disponibles

  • Libro: Representa un libro de la biblioteca, con atributos como título, autor y editorial.
  • Autor: Representa a los autores registrados en la biblioteca.
  • Editorial: Representa las editoriales asociadas a los libros.

Endpoints Disponibles

Libros

  • GET /api/v1/libros
    Devuelve una lista de todos los libros registrados.
  • GET /api/v1/libros/{id}
    Devuelve los detalles del libro con el ID especificado.
  • POST /api/v1/libros
    Crea un nuevo libro. (Enviar datos como JSON en el cuerpo de la solicitud)
  • PUT /api/v1/libros/{id}
    Actualiza los datos del libro con el ID especificado. (Enviar datos como JSON en el cuerpo de la solicitud)
  • DELETE /api/v1/libros/{id}
    Elimina el libro con el ID especificado.

Autores

  • GET /api/v1/autores
    Devuelve una lista de todos los autores registrados.
  • GET /api/v1/autores/{id}
    Devuelve los detalles del autor con el ID especificado.
  • POST /api/v1/autores
    Crea un nuevo autor. (Enviar datos como JSON en el cuerpo de la solicitud)
  • PUT /api/v1/autores/{id}
    Actualiza los datos del autor con el ID especificado. (Enviar datos como JSON en el cuerpo de la solicitud)
  • DELETE /api/v1/autores/{id}
    Elimina el autor con el ID especificado.

Editoriales

  • GET /api/v1/editoriales
    Devuelve una lista de todas las editoriales registradas.
  • GET /api/v1/editoriales/{id}
    Devuelve los detalles de la editorial con el ID especificado.
  • POST /api/v1/editoriales
    Crea una nueva editorial. (Enviar datos como JSON en el cuerpo de la solicitud)
  • PUT /api/v1/editoriales/{id}
    Actualiza los datos de la editorial con el ID especificado. (Enviar datos como JSON en el cuerpo de la solicitud)
  • DELETE /api/v1/editoriales/{id}
    Elimina la editorial con el ID especificado.

Detalles Técnicos

La API está implementada usando Spring Boot y MariaDB. Soporta formato JSON para las solicitudes y respuestas. Los endpoints están protegidos para evitar acceso no autorizado (si configuraste Spring Security).

Ejemplos de Peticiones

Para obtener todos los libros registrados en el sistema, realiza una petición GET a:

Libros: /api/v1/libros

Autores: /api/v1/autores

Editoriales: /api/v1/editoriales
